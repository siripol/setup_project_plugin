# `.claude/docs/` — load-on-demand context

Long-form conventions, references, and standards that Claude reads **only when the work touches the topic**. Different load behavior from `.claude/rules/`, which is always-on.

## Why load-on-demand

`CLAUDE.md` is loaded every turn, so it must stay small (identity + profile + policies table + pointers). Long reference material — API conventions, observability standards, downstream-resilience patterns — would bloat every prompt with content that is irrelevant most of the time. The `.claude/docs/` tier exists so the bodies stay searchable and discoverable without occupying always-on context.

Claude reads a file in this directory when:

- The user asks about a topic that maps to a doc here.
- A task touches code in an area the doc covers (e.g., editing an API handler triggers a read of an API-conventions doc).
- An always-on rule in `.claude/rules/` points at a doc here for the long-form version.

## Where bodies come from

Three populating sources:

| Source | Examples | Behavior |
|---|---|---|
| Profile overlay | `microservice-conventions.md`, `bff-aggregation.md` | Ship as template files in `templates/profile/<P>/claude/docs/` |
| Policy catalog | `policy-<slug>.md` per applied policy | Added on `sn-setup policy apply <slug>`; removed on `sn-setup policy remove <slug>` |
| Project-team-authored | Anything the team adds in a PR | Free-form; treat as code (PR-reviewed, CODEOWNERS-protected) |

## How to add a new body

1. Decide the slug: `kebab-case`, descriptive, distinct from existing bodies.
2. Author at `.claude/docs/<slug>.md`. Start with a one-sentence purpose paragraph (when Claude should read it). Then the body.
3. Cross-reference from `CLAUDE.md`'s `## Context policy` section or from an `.claude/rules/<slug>.md` pointer that triggers the load.
4. Open the PR. Reviewer checks: is the body load-on-demand-shaped (long-form, topical), not always-on-shaped (short, prescriptive)? If always-on-shaped, route to `.claude/rules/` instead.

## Relationship to `.claude/rules/`

| Tier | Where | Length budget | When loaded |
|---|---|---|---|
| Always-on minimal | `CLAUDE.md` | Whole file ≤ 500 tokens | Every turn |
| Always-on rules | `.claude/rules/<slug>.md` | ≤ 50 tokens each | Every turn |
| Load-on-demand | `.claude/docs/<slug>.md` | Up to ~3000 tokens per body | When topic surfaces |

If a piece of content is always-relevant + short, it belongs in `.claude/rules/`. If it is long, topical, and only relevant when work touches the topic, it belongs here.
